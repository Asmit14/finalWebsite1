1. What is the Hypertext Transfer Protocol (HTTP)
-> An application layer protocol used to transmit 'hypermedia' documents like HTML.

2. Why is HTTP called a stateless protocol?
-> It follows a client-server model and is a stateless protocol which means it does not retain any data in between requests from the client side.

3. Describe the three phases of establishing a session using client-server protocols such as HTTP
-> (1)The client establishes the connection through either a TCP connection or any other suitable transport layer.
(2)The client sends the request to the server.
(3)The server processes and sends back the answer while also providing a status code as well as any relevant data.

4. Describe the parameters on a get and post request.
-> GET : The parameter name and it's associated values are present.
POST: The keys and values are encoded in a data structure and non alphanumeric characters are percent encoded.

5. What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?
-> Response status codes indicate whether requests are successfully completed or not.
HTTP status code 200 - the request has succeeded.
HTTP status code 404 - the server was unable to find the requested resource.

6. What are HTTP cookies? Give an example.
-> It is data sent by the server to a client's web browser such that the browser may store it and send it back with rewuests to the same server at a later point in time.
the option to stay signed in or to remember passwords during login to online services are the result of cookies.

7. Why are cookies a security issue?
-> They can be read an modified/edited by those who have access to a clients hard disk in spite of the security measures taken to ensure cookies cannot fall into the hands of
third parties and only go to the intended recipient.

8. Describe session hijacking and XSS
-> XSS, referring to cross site scripting is a security gap which allows an attacker to inject malicious client side code into the website. This code is later executed by the
victims of the attack and allow the attacker to get past security/access controls and subsequently impersonate users.
Session Hijacking relies on an attackers knowledge of a users session cookie. Once a user is authenticated on the server side, the attacker is capable of using the session cookie to
hijack the session on their own browser.

9. Describe Cross-site request forgery(CSRF)->
-> An attacker can use XSS to launch a CSRF attack. The attacker causes a user's browser to send a request to the website without the user's own knowledge.

10. What are Cross-site HTTP requests (CORS) and what requests use CORS?
-> They are requests for resources belonging to a different domain than the domain of the resource that makes the request.
CSS Shapes from images makes such requests.
